\subsection{GRPO background and algorithm overview}

Group Relative Policy Optimization trains a language model from scalar rewards assigned
to complete sampled responses. For each prompt, it generates several responses, compares
their rewards, and uses the relative result to change the probabilities of the sampled
token actions. The central training unit is therefore a group: one prompt and multiple
sampled response trajectories.

Let $q$ denote a prompt and let
$o_i=(o_{i,1},\ldots,o_{i,T_i})$ denote sampled response $i$. GRPO performs the following
high-level loop:
\begin{enumerate}
  \item freeze the current model and use that snapshot to sample $G$ responses to the
        same prompt;
  \item assign one reward to each complete response;
  \item compare rewards within the group to obtain one advantage per response;
  \item replay the sampled tokens through the trainable policy;
  \item increase the probabilities of actions from positive-advantage responses and
        decrease those from negative-advantage responses;
  \item clip large changes relative to the frozen sampling snapshot and penalize drift
        from a separate frozen reference model; and
  \item update the model and generate a new rollout group.
\end{enumerate}

The next modules unpack these steps in exactly this order. The initial explanation assumes
a synchronous learner and a simple response containing only model-generated tokens.
Rollout buffers, asynchronous workers, and agentic trace construction are introduced only
after the basic update is established. DeepSeek-R1 uses GRPO during its reinforcement-
learning stages \citep{guo2025deepseekr1}; this attribution identifies the source studied
here but is not needed to understand the algorithm.
