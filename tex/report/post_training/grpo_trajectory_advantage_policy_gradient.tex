\subsection{The gradient direction behind the GRPO reward term}
\label{subsec:grpo-gradient-direction}

The reward-driven part of the complete loss in Equation~\ref{eq:grpo-full-loss} contains
the product $\rho_{i,t}A_i$. Before deriving the ratio $\rho_{i,t}$, it is useful to expose
the gradient direction that this product must produce. For that purpose only, define the
teaching surrogate
\begin{equation}
  \widetilde J_i(\theta)
  = A_i\frac{1}{T_i}\sum_{t=1}^{T_i}
    \log\pi_\theta(o_{i,t}\mid q,o_{i,<t}),
  \qquad
  \widetilde{\mathcal{L}}_i=-\widetilde J_i.
  \label{eq:grpo-basic-trajectory-objective}
\end{equation}
This is not an additional term to be added to the GRPO loss. It is a temporary expression
whose gradient makes the effect of $A_i$ easy to see. Replay sampled trajectory $i$
through the trainable model with teacher forcing and retrieve the log-probability of every
sampled token.

The advantage is computed before optimization and treated as a detached constant. Its
effect becomes clear by differentiating:
\begin{equation}
  \nabla_\theta \widetilde J_i
  = A_i\frac{1}{T_i}\sum_{t=1}^{T_i}
    \nabla_\theta\log\pi_\theta(o_{i,t}\mid q,o_{i,<t}).
  \label{eq:grpo-advantage-weighted-gradient}
\end{equation}

If $A_i>0$, gradient ascent on $\widetilde J_i$ increases the log-probabilities of the
sampled actions. Because training software minimizes
$\widetilde{\mathcal{L}}_i=-\widetilde J_i$, the optimizer produces the same direction. If
$A_i<0$, the sign reverses and the sampled actions become less likely. If $A_i=0$, the
trajectory contributes no reward-driven gradient.

For a positive-advantage response `5 <eos>', the sampled `5' is reinforced under the
original prompt and the sampled end token is reinforced under the prefix containing `5'.
For a negative-advantage response `4 <eos>', both sampled decisions are suppressed. The
same trajectory advantage is broadcast over the token positions, but each token has a
different conditional probability and therefore a different parameter gradient.

The Transformer parameters are shared across all positions and examples. Training does
not independently edit a stored probability for each token. The batch loss adds the
weighted gradients from every sampled action; those contributions may agree or conflict,
and one parameter update changes behavior in many unobserved contexts. The factor
$1/T_i$ gives each trajectory equal total normalization. A global mean over all valid
tokens instead gives longer responses more influence, so the reduction rule must be stated
when trajectories have different lengths. Section~\ref{subsec:grpo-importance-ratio} now
returns to the actual
$\rho_{i,t}A_i$ term in Equation~\ref{eq:grpo-full-loss} and shows why its gradient matches
Equation~\ref{eq:grpo-advantage-weighted-gradient} at the behavior policy.
