\subsection{Polar: preserving on-policy unit traces}

Polar places a proxy at the model API boundary and records the exact token-level calls made
during an agent session \citep{xu2026polar}. Assistant tokens are policy actions. They are
trainable only when they were sampled by the recorded behavior policy. Prompts, tool
responses, and other external context condition those actions but receive zero loss.

Polar's conservative strategy treats every model request as a separate on-policy unit
trace. Its prefix-merging strategy forms a longer trace only when a grouping key marks a
continuation and the next request preserves the preceding token sequence as an exact
prefix. In that case, the later action continues the same sampled trajectory.

Context compaction, prompt replacement, a new subagent, or a parallel branch breaks the
prefix relation. The new request must initialize a new unit trace rather than be treated as
the next state of the old trace. Otherwise the trainer would combine actions across an
off-policy boundary and the recorded behavior-policy probabilities would no longer
describe one valid sampled chain.

Within every unit trace, sampled assistant tokens retain their original token IDs and
behavior-policy log-probabilities. Tool responses and other required context remain visible
to the policy but are masked from the loss. The invariant is simple: every trainable token
must be an exact action sampled by the behavior policy associated with that trace; when
this can no longer be guaranteed, start a new trace.
