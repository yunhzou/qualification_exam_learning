\subsection{From hidden states to token probabilities}

After the final decoder block, the model normalizes each hidden vector and applies the
language-model head:
\begin{equation}
  Z=H^{(L)}W_{\mathrm{vocab}},
  \qquad W_{\mathrm{vocab}}\in\mathbb{R}^{D\times V},
  \qquad Z\in\mathbb{R}^{B\times T\times V}.
\end{equation}
Each row of $Z$ contains one score, or logit, for every possible next token. Softmax turns
these scores into probabilities. The head contains $DV$ parameters and costs about
$2BTDV$ FLOPs. Tying $W_{\mathrm{vocab}}$ to the input embedding table avoids storing a
second $VD$ parameter matrix, but it does not remove this projection's computation.

The $BTV$ logits can be large. Fused or chunked head, softmax, and cross-entropy operations
avoid retaining unnecessary probability tensors. Autoregressive decoding needs a
vocabulary distribution only at the final position, reducing this step's output from
$[B,T,V]$ to $[B,V]$.
