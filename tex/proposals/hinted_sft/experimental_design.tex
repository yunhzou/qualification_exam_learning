\section{Minimal experiment}

The first experiment should be deliberately small. Collect agent trajectories with one
localized, diagnosable mistake: an unavailable tool call, a missed constraint, or a tool
result that should change the plan. For each trajectory, place one short corrective hint
at the point where the model should have realized the issue.

\paragraph{Compared methods.}
\begin{enumerate}
  \item \textbf{OPSD:} provide the hint only to the teacher and apply KL to the student's
  later token distribution.
  \item \textbf{Hinted SFT:} apply cross-entropy only to the hint tokens. Mask the entire
  original trajectory and the entire continuation.
\end{enumerate}

Both methods use the same trajectories and the same human- or model-written hints. This
isolates the only intended difference: OPSD trains the distribution after an unobserved
hint, while hinted SFT trains the model to produce the hint itself.

\paragraph{Evaluation.}
Measure task success after the mistake, whether the model generates the correct hint at
the correct point, and whether the later action follows the generated hint. A direct
causal check is to remove the generated hint from the prefix before continuing. If the
later behavior does not change, the hint was not actually driving the continuation.

The experiment should also include counterexamples where the proposed hint depends on
information absent from the trajectory. Hinted SFT should not be trained on those
examples. Their purpose is to show the boundary of the claim: the method teaches a model
to articulate an inference available from its context, not to reconstruct hidden
information.
