\section{Hinted SFT algorithm}

The algorithm requires no new loss and no special inference mechanism.

\begin{enumerate}
  \item Sample or collect an agent trajectory containing a localized mistake.
  \item Choose the point where the model should have noticed the mistake.
  \item Write a short hint $h^\star$ using only information available in the trajectory.
  \item Insert $h^\star$ as ordinary assistant thinking immediately before the corrected
  continuation.
  \item Apply SFT loss only to the hint tokens. Mask every token before and after the hint.
\end{enumerate}

For a trajectory prefix $x$, the core objective is simply
\[
  \mathcal L_{\mathrm{hint}}
  =-\frac{1}{|h^\star|}\sum_j
    \log \pi_\theta(h^\star_j\mid x,h^\star_{<j}).
\]
System, user, tool-result, trajectory-history, and continuation tokens all have mask
$m=0$. Only the hint tokens have $m=1$. After the model generates the hint, later tokens
are decoded normally from
$\pi_\theta(\cdot\mid x,h)$.

  \caption{\textbf{Training and deployment use the same sequence.} Training supplies the
  hint token target and masks everything else; deployment lets the model generate those

\end{figure}

The main data requirement is also simple: do not train hints that depend on information
the deployed model cannot access. If the required evidence is missing, the correct thought
is to retrieve it or state uncertainty, not to guess the privileged answer.
