\begin{abstract}
On-policy self-distillation (OPSD) can insert a corrective hint into a teacher context and
pull an unhinted student distribution toward the resulting continuation. The student,
however, never generates the hint that caused the teacher's behavior. This can be
undesirable when the hint expresses a diagnosis or realization that should be part of the
model's own reasoning. The proposed alternative is simple: put the hint into the training
trajectory as ordinary thinking tokens and use supervised fine-tuning (SFT) to teach the
model to generate them. During deployment there is no additional component. The model
thinks normally, generates the learned hint when the relevant situation occurs, and then
conditions its later tokens on that hint because it is already part of the autoregressive
prefix. This proposal compares that \emph{hinted SFT} construction with OPSD on localized
errors in long agent trajectories.
\end{abstract}

\section{Research question and position}

\paragraph{Research question.}
When a short diagnosis or reminder can be inferred from the model's current trajectory,
is it better to train the model to generate that thought explicitly than to pull its
unhinted continuation distribution toward a teacher that was given the thought?

OPSD uses the policy with its original context as the student and the same or a related
policy with a privileged hint as the teacher. A KL loss moves the student's token
distribution toward the hint-conditioned teacher
\citep{shenfeld2026selfdistillation,zhao2026selfdistilledreasoner,cursor2026composer25}.
Composer 2.5 uses this mechanism for localized textual feedback such as reminding an
agent which tools are available after a bad tool call \citep{cursor2026composer25}.

That method can work: parameter updates may compile the effect of the hint into the
unhinted policy. The concern is narrower. The teacher's continuation was caused by a hint
that the deployed student never sees and never learns to produce. If the hint contains a
useful intermediate diagnosis, directly matching later token probabilities does not teach
the model the explicit reasoning step that produced those probabilities. If the hint also
contains information that cannot be inferred from the student's context, the target may
encourage an unsupported guess or hallucination.

Hinted SFT moves the supervision to the missing step. Given a trajectory prefix $x$ and a
short validated hint $h^\star$, train the model to generate $h^\star$ as its next thinking
tokens. Those tokens then remain in context and naturally change the continuation. There
is no special deployment pipeline, reviewer, second model, or hint injector. The deployed
model simply generates its normal autoregressive reasoning.

The proposal therefore applies only when the hint is supported by information already in
the trajectory or obtainable through the normal agent interface. If the hint contains a
hidden answer or unavailable future information, hinted SFT does not solve the problem;
that example should not be used.
