\section{The missing-thinking-step problem}

Figure~\ref{fig:opsd-versus-hinted-sft} shows the entire idea. In OPSD, the hint changes
the teacher distribution, and KL transfers that change to the original-context student.
In hinted SFT, the hint itself is the supervised target. Once generated, it becomes part
of the student's own context and causes the later distribution shift.

\paragraph{Concrete example.}
An agent attempts to call tool C, but the system prompt lists only tools A and B. A
reviewer writes the hint: \emph{Tool C is unavailable; use A or B.} OPSD inserts that hint
only into the teacher context and pulls the student's next-action distribution toward A
or B. Hinted SFT instead trains the model to generate that sentence in its thinking after
the failed call. At deployment, the model generates the realization itself; the tokens
\emph{use A or B} are now in context, so the later action distribution changes for an
ordinary autoregressive reason.

This difference matters most when the thought is reusable across several later steps.
A localized hint can update the model's current plan, not just the probability of one
immediate token. It also creates an inspectable intermediate: one can check whether the
model noticed the error before it acted again.
